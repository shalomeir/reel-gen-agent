# PRD: reel-gen-agent

이 프로젝트의 핵심 정본 문서다. 무엇을 만들고, 누구를 위해, 어디까지 이번에 만들고,
무엇으로 성공을 판단하는지를 정한다. `specs/project-brief.md`가 상위 비전이라면 이 문서는
그 비전을 구현 기준으로 푼 요구사항이다. 구현은 이 문서와 `specs/`의 하위 설계만 따른다.
`docs/`는 배경 자료이지 구속력이 없다.

작성일: 2026-06-29. 상태: 초안(스코프 확정 진행 중).

## 1. Overview

영상 팀 없이 한 사람이, 잘 만든 레퍼런스 숏폼의 리듬을 그대로 빌려 바로 올릴 수 있는
세로 광고 클립을 뽑는다. 스타일은 베끼는 게 아니라 측정해서 데이터로 재현한다. 레퍼런스를
분석한 엔진이 결과물도 같은 잣대로 채점하므로, "이 레퍼런스처럼"이 느낌이 아니라 점수로
확인된다.

## 2. 제품 목적과 설명 (Problem / Solution)

### Problem

여성 라이프스타일 소비재를 다루는 DTC 브랜드 마케터에게 숏폼 광고는 성과의 핵심
채널이지만, 한 편을 만들 때마다 모델, 촬영, 편집, 음악, 자막을 직접 챙겨야 한다. 잘 되는 레퍼런스를 봐도 왜
잘 되는지(컷 속도, 음악 템포, 첫 3초 훅, 톤)를 손으로 분해해 다른 제품에 옮기기 어렵다.
기존 생성 도구는 스타일이 고정 프리셋이라 특정 레퍼런스의 편집 리듬을 재현하지 못하고,
제품과 모델 일관성도 매 컷 흔들린다.

정리하면 세 가지가 막혀 있다.

- 좋은 레퍼런스의 "왜 잘 되는지"를 재사용 가능한 형태로 뽑아내지 못한다.
- 그 리듬을 새 제품에 옮겨 영상으로 만드는 자동 경로가 없다.
- 만든 결과가 레퍼런스만큼 좋은지 객관적으로 판단할 기준이 없다.

### Solution

레퍼런스 영상을 재사용 가능한 프로필 JSON(`VideoProfile`)로 바꾼다. 컷 리듬, 오디오
다이내믹, 색은 로컬 라이브러리가 수치로 재고, 톤, 자막 스타일, 훅, 내러티브 아크는 Gemini가
읽는다. 이 프로필을 씨앗 삼아 새 제품용 스토리보드와 영상을 생성한다.

사람은 제품, 모델(캐릭터), 영상 목적(컨셉) 세 가지만 확정한다. 이 세 입력이 들어오면
시스템은 가장 먼저 첫 3초 훅을 만든다. 훅 생성이 이 파이프라인의 핵심 단계다. 영상 목적이
무엇을 약속하고, 어떤 제품을, 누가 보여주는지를 한 문장과 한 비주얼로 압축해 시청을 멈추게
하는 장면을 짠다. 같은 제품이라도 목적과 캐릭터가 바뀌면 훅이 달라져야 하고, 훅이 정해지면
그 뒤의 콘티, 패널 타이밍, 자막 카피가 모두 그 훅을 받쳐 흐른다. 나머지(콘티 구성, 패널
프롬프트, 에셋과 스틸 생성, 켄번스 조립, 배경음악, BGM 템포와 컷 템포 정렬)는 에이전트가
자동으로 만든다. 생성이 끝나면 같은 분석기가 결과물을 다시 프로파일링해 목표 스타일과
비교 채점한다. 분석기 하나가 프로파일러이자 채점기로 동작해, 레퍼런스와 생성물이 한 잣대
위에 선다.

배포 형태는 오픈소스 CLI다. 자기 API 키만 있으면 누구나 자기 모델 사용료만 내고 돌린다.
호스팅도 계정도 없다.

## 3. 주요 Target / ICP

핵심 사용자(ICP)는 두 부류다. 여성 라이프스타일 소비재 DTC 브랜드의 마케터, 그리고
그런 마케터를 위해 콘텐츠를 만드는 개발 친화적인 크리에이터다. 둘 다 주요 대상이다.

- **누구**: 여성 라이프스타일에 어필하는 소비재(뷰티 중심) DTC 브랜드의 마케터. 주
  소비자가 10~30대 여성인 제품을 다룬다. 그리고 이런 브랜드의 콘텐츠를 대신 제작하는
  크리에이터(개발을 이해하는 제작자, 소규모 에이전시 포함)도 같은 핵심 대상이다.
- **기술 수준**: 터미널에서 CLI를 돌리고 API 키를 발급할 수 있는 정도. 개발자거나
  개발 친화적인 마케터, 또는 개발을 이해하는 콘텐츠 제작자.
- **제품군**: 한 사람이 실내에서 몸에 걸치거나 써 보일 수 있는 것. 빈도순으로 스킨케어
  (가장 큼), 메이크업, 인접 뷰티와 웰니스(영양제, 시술, 이너웨어, 애슬레저), 확장 영역
  (의류, 액세서리, 가방, 신발, 안경, 간단한 홈 데코). 기본 모델과 팔레트와 보이스는
  여성향으로 잡는다. 숏폼 제품 PPL이 거기에 몰려 있다.
- **동기**: 영상 팀 없이도, 잘 되는 레퍼런스 리듬에 맞춰 여러 변형을 빠르고 싸게
  찍어내 광고 성과를 올리고 싶다.

맞지 않는 사용자: 영상 한 편을 손으로 끝까지 다듬고 싶은 사람, 중장비 같은 산업재나
실내에서 보여줄 수 없는 제품을 다루는 마케터.

## 4. 유저 핵심 시나리오

1. **레퍼런스 한 편을 닮은 첫 클립.** 세럼 하나와 빠른 컷 UGC 레퍼런스를 넣는다. 모델과
   제품 아트, 콘티를 확인하면, 키워드 자막과 경쾌한 베드가 깔린 15초 세로 클립이 나온다.
   그날 바로 올린다.
2. **같은 제품, 다른 리듬 셋.** 같은 입력에 레퍼런스만 빠른 UGC, 느린 클리니컬 데모,
   시네마틱 브랜드 필름으로 바꿔 돌린다. 편집 리듬이 뚜렷이 다른 세 결과가 나와 A/B로
   본다.
3. **원샷 런 모드.** 기본값을 믿는 사용자가 `run`으로 돌리면, 입력 파일 하나에서 중간
   확인 없이 완성 mp4를 받는다.

## 5. 유저가 경험할 Key Success Event

사용자가 "이거다" 하고 느끼는 순간은 두 가지다.

- **첫 성공**: 핵심 입력 세 개(제품, 모델, 컨셉)만 chat으로 확인했는데, 자기 제품이 레퍼런스
  릴의 컷 리듬과 음악 템포에 맞춰 움직이는 클립이 나오고, 손대지 않고 올릴 만한 수준일 때.
- **재현 체감**: 같은 입력에 레퍼런스만 바꿨더니 편집 리듬과 분위기가 확 달라질 때.

이 두 경험이 제품의 약속이다. 성공지표(8장)는 이 둘을 수치로 옮긴 것이다.

## 6. 기능적 요구사항

### 6.1 Key Features

- **레퍼런스 분석(완성)**: 영상 한 편을 `VideoProfile` JSON으로. 정형 계층이 컷 리듬,
  오디오 다이내믹, 색을 재고, Gemini 계층이 톤, 자막, 훅, 내러티브 아크를 읽는다.
- **분석에서 생성 입력 시딩**: 패널 수와 패널별 타이밍을 `style_profile`의 컷 데이터에서
  뽑는다. 빠른 컷 레퍼런스는 짧은 패널을 많이, 느린 컷 레퍼런스는 긴 패널을 적게 만든다.
- **단계별 그래프**: 생성 파이프라인을 LangGraph 노드로 단계 분리한다. 그래프 자체는 단계별
  사람 확인 게이트 없이 입력 -> ReelProfile -> production을 일괄로 돈다(run). 단계별 HITL
  게이트는 향후 과제다([../retro.md](../retro.md)).
- **대화형 확인(chat)**: `chat`은 대화로 핵심 입력(제품, 모델, 영상 목적)을 채운 뒤 ReelProfile과
  대표이미지를 보여주고, 확인·수정 루프를 돌려 확인하면 production으로 넘어간다. `run`은 이
  확인 없이 목적만으로 나머지를 추론해 끝까지 돈다.
- **훅 생성기(핵심 단계)**: 핵심 입력 세 개(영상 목적, 제품, 캐릭터)를 받아 첫 3초
  훅을 만드는, 생성 파이프라인의 중심축이다. 좋은 훅 레퍼런스를 사전 지식으로 두고, 패턴을
  완전히 규격화하지 않은 채 가이드라인만 준다. 모델(Claude)이 세 입력을 헤드라인, 제품 한
  줄, 비주얼 훅으로 압축하며 비결정적이다(temperature). 같은 입력에서도 다른 훅이 나오고,
  여기서 정해진 훅이 콘티와 패널 카피의 기준이 된다.
- **에셋 바이블**: 캐릭터와 제품의 일관성을 다각도 시트 이미지로 먼저 고정한다.
- **스토리보드**: global 프롬프트(캐릭터, 제품, 컬러, 무드 공통)와 패널별 local 프롬프트로
  나눠 콘티를 만든다. affordance 목록과 컨셉 템플릿을 사전 지식으로 주입해 콘티 품질을
  높인다.
- **켄번스 조립**: 패널 스틸에 카메라 모션을 걸어 ffmpeg로 잇고, 자막(pilmoji 컬러 이모지
  오버레이)과 배경음악을 얹어 mp4로 만든다. 영상 모델 예산 없이도 끝까지 돈다.
- **리듬 정렬**: 배경음악의 템포와 컷 편집 템포를 맞춘다. BGM 비트에 패널 경계를 스냅한다.
  (voice 자체의 템포는 정렬 대상이 아니다.)
- **닫힌 루프 score 체크**: 생성물을 같은 분석기로 다시 프로파일링해 목표 스타일 프로필과
  비교 채점한다.

### 6.2 MoSCoW (이번 구현 기준)

- **Must (이번 핵심)**
  1. 분석 → `generation_input` 시딩(컷 리듬)
  2. LangGraph 노드 그래프(plan/execute, 단계별 사람 확인 게이트 없음)
  3. chat 대화형 확인·수정 루프 + run 일괄 실행
  4. 훅 생성기(레퍼런스 기반 가이드라인, 비결정)
  5. 에셋 바이블(캐릭터 + 제품)
  6. 스토리보드(global + local 프롬프트, affordance와 템플릿 사전 지식)
  7. 켄번스 조립 + 자막(pilmoji) + 배경음악 mux
  8. 리듬 정렬(BGM 템포 ↔ 컷 편집 템포)
  9. 닫힌 루프 score 체크(golden 대비)
- **Should (보조)**
  - fal.ai 영상 백엔드(Seedance, Kling 등)로 켄번스 경로를 교체(이미지는 Nano Banana 단일 경로 유지)
  - 플랫폼 제약 자동 주입(종횡비, 길이 상한, 세이프 존, 자막 가독성)
  - voiceover 옵션(기본 off, 켜면 비트 동기는 하지 않고 패널 타이밍에만 맞춤)
- **Won't (이번 제외)**
  - VLM 품질·안전 검사와 샷 단위 자동 재생성
  - 컨셉 템플릿 대량 확장
  - 호스팅 SaaS, 계정, 결제, GUI

## 7. 비기능 요구사항 (NFR)

- **지연/성능**: 로컬 CLI로 노트북에서 돈다. chat의 대화 응답은 즉시 온다. run당 총 생성
  시간에 상한 목표를 둔다(구체 수치는 열린 항목, `specs/eval.md`에서 확정).
- **가용성/복원력**: 외부 모델 API에 의존하므로 호출은 재시도를 건다(tenacity). 단계가
  실패하면 그 단계만 격리하고 재개할 수 있어야 한다. 켄번스 폴백이 있어 영상 모델 없이도
  파이프라인이 완주한다.
- **비용 의식**: 이미지와 분석 기본은 flash급 모델. run당 비용에 상한 목표를 둔다(열린
  항목). chat은 비싼 영상 단계 전에 ReelProfile 확인·수정 루프로 한 번 끊어 준다(run은 무중단).
- **보안**: 키는 `.env`에만 두고 커밋하지 않는다. `.env.example`은 키 이름과 발급처와
  용도만 적는다. 입력은 검증한다. 생성물은 브랜드 훼손, 로고, 민감 표현을 점검한다(안전
  검사 자체의 자동 재생성은 이번 제외, 8장 Guardrail로 측정).
- **재현성**: `outputs/<run_id>/`에 입력, 에셋, 스토리보드, 패널 스틸, 최종 mp4를 보존한다.
  결정적 계층(분석 수치)은 재현 가능하다.
- **이식성**: 코어는 CLI가 인프로세스로 부르는 패키지다. 나중에 웹 계층이 같은 코어를
  감쌀 수 있게 둔다.

## 8. 성공지표 (North Star + Guardrail)

- **North Star**: 한 번의 run으로 "레퍼런스 스타일을 닮은 post-ready 세로 클립"을 뽑아내는
  비율. 닮음의 정의는 생성물을 같은 분석기로 다시 프로파일링한 style-match score가 골든
  기준선(14장 Eval)을 통과하는 것이다.
- **Guardrail (North Star를 쫓다 무너뜨리면 안 되는 것)**
  - 캐릭터와 제품 일관성 깨짐 0건
  - 브랜드, 로고, 안전 위반 0건
  - run당 비용 상한 준수
  - run당 생성 시간 상한 준수

style-match를 끌어올리려다 일관성이나 안전이나 비용이 무너지면 그 run은 실패로 본다.

## 9. 용어 정의

- **VideoProfile / style_profile**: 영상 한 편을 잰 프로필 JSON. 분석 산출물이자 생성의
  씨앗이자 채점 기준.
- **generation_input**: 컨셉을 생성용 파라미터로 직렬화한 입력 JSON.
- **asset bible(에셋 바이블)**: 캐릭터와 제품의 일관성을 고정한 레퍼런스 이미지 묶음.
- **storyboard / panel**: 컷 단위 콘티와 그 한 컷. 패널은 타이밍, 카메라, 자막, 스틸을
  담는다.
- **global / local prompt**: 모든 컷에 공통으로 흐르는 맥락(global)과 컷별 동작 지시(local).
- **chat / run**: 사람 확인 방식. chat은 대화로 입력을 채우고 ReelProfile을 확인·수정한 뒤
  생성, run은 확인 없이 입력에서 끝까지 일괄 생성.
- **HITL**: Human In The Loop. 사람이 확정해야 넘어가는 지점. 그래프 안 단계별 HITL 게이트는
  이번 범위 밖이다(향후, [../retro.md](../retro.md)).
- **affordance**: 제품이 가능케 하는 행동 목록(예: 스프레이는 흔든다, 뿌린다).
- **narrative_arc / template**: 이름 있는 컨셉 구성 규칙(UGC, before/after, 튜토리얼 등).
- **hook**: 첫 3초의 시선 고정 장치(헤드라인, 제품 한 줄, 비주얼 훅).
- **golden set**: 분석 점수가 S급, A급으로 판정된 레퍼런스 영상 집합. 평가 기준선.
- **style-match score**: 생성물 프로필을 골든 기준과 비교한 유사도 점수.
- **켄번스(Ken Burns)**: 스틸 이미지에 천천히 줌과 팬을 걸어 움직임을 만드는 기법.

## 10. 예상 Risk 및 종속성

### Risk

- **외부 모델 의존**: 이미지, 영상, 음악 API의 사양 변경, 비용 상승, 레이트 리밋. 완화로
  스키마 뒤에 백엔드를 두어 교체 비용을 한 단계로 가둔다.
- **캐릭터와 제품 일관성**: 이미지 모델 한계로 컷마다 얼굴이나 패키지가 흔들릴 수 있다.
  에셋 바이블을 먼저 고정하고 패널 스틸에 레퍼런스 이미지를 함께 넘겨 완화한다.
- **리듬 정렬 난이도**: BGM 비트 검출 정확도와 컷 스냅의 자연스러움. 결정적 비트 검출
  (librosa)로 시작하고, 어긋남은 chat 확인·수정 루프에서 사람이 잡는다.
- **영상 모델 길이 제한**: 한 번에 짧은 길이만 만드는 모델이 많다. 패널을 한계에 맞게
  쪼개 생성하고 ffmpeg로 잇는다.
- **평가의 주관성**: 좋은 영상의 기준은 사람마다 다르다. 분석기 점수로 객관화하되, 그
  점수가 사람 판단을 완전히 대신하지는 못한다는 한계를 받아들인다.

### 종속성

- **필수**: ffmpeg, Gemini API(분석과 이미지 생성).
- **Should**: fal.ai(영상 모델), 음악 생성 모델, pilmoji와 컬러 이모지 폰트.
- **옵션**: 보이스오버용 TTS.

## 11. 벤치마킹할 서비스

- **higgsfield**: 메인 벤치마크. 제품과 캐릭터와 컨셉을 사람이 확인하고 나머지를 자동
  생성하는 구성(우리는 chat 대화형 확인으로 대응), 에셋 일관성 고정 방식을 참고한다.
- **AI 숏폼 광고 생성 카테고리 일반**: UGC형 광고 자동 생성 도구들과 비교군으로 둔다.
  차별점은 "고정 프리셋이 아니라 레퍼런스에서 측정한 스타일을 재현하고, 같은 엔진으로
  결과를 채점하는 닫힌 루프"다.

## 12. 주요 사용자 유스케이스 (입력 → 확인 → 출력)

사람 확인은 chat의 확인·수정 루프(ReelProfile 단계에서 한 번)로 하거나, run으로 생략한다.

| 유스케이스 | 입력 | 사람 확인 | 출력 |
| --- | --- | --- | --- |
| 첫 클립 생성 | 제품 정보, 레퍼런스 영상/프로필 | chat: ReelProfile 확인·수정 | 15초 세로 mp4 + 중간 산출물 |
| 리듬 변형 비교 | 같은 제품, 레퍼런스 3종 | chat 3회 또는 run 3회 | 편집 리듬이 다른 3편 |
| 카테고리 확장 | 비뷰티 제품, 착용 레퍼런스 | chat: ReelProfile 확인·수정 | 착용 시퀀스 세로 mp4 |
| 런 모드 | `generation_input.json` | 없음(무중단) | 무중단 완성 mp4 |
| 결과 점검 | 생성물 mp4 | 없음 | style-match score와 등급 |

## 13. Agent Capability & Boundary

에이전트가 스스로 결정하고 실행하는 범위와, 사람이 확인하는 지점을 명시한다.

### 자율 실행 (기본 auto flow)

- **훅 생성(핵심 단계)**: 핵심 입력 세 개(제품, 모델, 영상 목적)가 정해지면 그 입력으로 첫
  3초 훅을 만든다. 이후 모든 자동 단계가 이 훅을 받쳐 진행된다.
- 레퍼런스에서 컨셉 잡기
- affordance 추출, 컨셉 템플릿 선택
- 스토리보드 패널 구성과 타이밍 산정(분석 컷 데이터 시딩)
- global과 local 프롬프트 작성
- 에셋 시트와 패널 스틸 생성
- 켄번스 조립, 배경음악 선택/생성, BGM 템포와 컷 템포 정렬
- 닫힌 루프 score 채점

### 핵심 입력 (chat에서 확인)

- **제품 정의**: 무엇을 파는지, USP, 패키지.
- **모델(캐릭터) 정의**: 누가 등장하는지.
- **영상 목적(컨셉)**: 무엇을 위한 영상인지, 어떤 톤과 내러티브인지.

chat 모드는 이 셋을 대화로 채우고, ReelProfile과 대표이미지로 묶어 한 번 확인·수정받은 뒤
생성으로 넘어간다. 그래프 안 단계별 사람 확인 게이트는 두지 않는다(HITL은 향후 과제,
[../retro.md](../retro.md)).

### Run mode

run은 위 확인 없이 입력(영상 목적 필수)만으로 나머지를 추론해 끝까지 무중단 생성한다.
대화형 확인이 필요하면 chat을 쓰지만 필수는 아니다.

## 14. Eval 기준

- **골든셋**: reference 영상들을 분석기로 점수화해, S급과 A급으로 판정된 영상을 기준선으로
  삼는다. 레퍼런스는 계속 추가 정리한다.
- **생성물 평가**: 생성물을 같은 분석기로 프로파일링한 뒤 골든 기준과 비교해 style-match
  score를 내고, S / A / B / C / D 등급을 매긴다.
- **평가 차원**
  - 컷 리듬 일치(패널 수, 컷 길이 분포)
  - 오디오와 리듬 동기(BGM 템포와 컷 템포 정렬)
  - 팔레트와 톤 일치
  - 훅 존재와 품질
  - 캐릭터와 제품 일관성
  - 안전(브랜드, 로고, 민감 표현)
- **열린 항목**: S / A / B / C / D 등급의 수치 컷오프, run당 비용과 시간 상한은 별도
  `specs/eval.md`에서 확정한다. 이 PRD는 기준 골격만 둔다.
